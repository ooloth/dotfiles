## Context

- All changes: !`git diff HEAD`
- Staged changes: !`git diff --staged`
- Unstaged changes: !`git diff`

## Your task

Please commit all changes in this working tree yourself. Then go back to deferring all future committing to me.

Start by running all checks and all tests. Fix any errors you encounter.

Then, divide the changes into multiple commits if relevant. Divide changes thematically. For example, pair one test case with its implementation and corresponding documentation updates. Changes in one file don't necessarily all belong in the same commit.

Ideally, sequence the commits in an order that progresses from building blocks to their integration. But do not revert and reply changes to force that sequence. Just do your best to show reviewers multiple small changes rather than a single large one when that would help their understanding.
