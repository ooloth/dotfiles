---
description: Commit changes in a logical sequence.
---

Please commit all changes yourself. Use multiple commits if relevant. Pair test cases with their implementation. Commit in a sequence that progresses from building blocks to orchestration.

Changes made to a single file don't necessarily belong in a single commit if their reason for changing was different. Look for "why?" boundaries.

Do not revert and reply changes to force the ideal sequence; that's going to far. Just do your best to divide the changes you see into a helpful progression.
